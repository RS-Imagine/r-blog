一键清理无用图片功能

在后台增加了一键清理无用图片的功能

成功：一键清理无用图片功能

已经按照计划在后台增加了一键清理无用图片的功能，现在可以毫无顾虑地上传图片了！

具体更新内容

1. 清理路由机制

在 admin/src/main.rs 中新增了一个专门负责清理工作的 API POST /api/cleanup-images：

  • 该接口会主动读取所有 .md 文章文件（从 content/posts/ 目录下）。
  • 它会使用简单的字符串扫描寻找任何类似 /images/xxxx.png 格式的引用链接，并提取文件名，记录在内存的白名单里。
  • 接着，它会检查 content/static/images/ 目录中所有的物理文件，如果在上述白名单里没有找到该文件名，程序就会果断且安全地把它从磁盘上删除掉，以此释放储存空间。

2. 界面按钮

在后台的首页面板 (dashboard) 中：

  • 在原有的 “New Post” 和 “Rebuild Site” 旁边，添加了一个红色的 “Clean Up Images” 按钮。
  • 出于安全考虑，这个按钮被按下去时会首先弹出一个系统确认框，只有点击“确认”以后，才会提交清理请求，避免由于手滑造成意外删除。

如何测试使用？

  1. 确保后台程序已运行：cargo run --bin admin
  2. 上传两张图片到任何文章中（或者甚至不用保存文章，直接在界面里点击上传图片）。
  3. 保存或离开页面，确保这之中至少有一张图片没有以 ![xxx](/images/xxx.png) 的形式保留在最终的 .md 文件正文里。
  4. 返回主控制面板，点击 Clean Up Images 按钮并确认。
  5. 会看到一个新的成功提示页面，上面会清晰地标出刚才删除了多少张没被使用的垃圾图片！
